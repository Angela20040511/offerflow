一 接口总规范
1 接口前缀
全部接口统一使用：
/api
2 认证方式
除登录注册外，全部接口都要求请求头携带 token。
Authorization: Bearer {token}
3 请求格式
GET
用于分页查询、详情查询、筛选查询。\n参数放在 query 中。
示例
GET /api/jobs?pageNum=1&pageSize=10&keyword=vue&city=上海
POST
用于新增、登录、投递、收藏、复杂查询。\n参数放在 JSON 请求体中。
示例
{\n  "username": "student01",\n  "password": "123456"\n}
PUT
用于更新。
DELETE
用于删除。
4 统一返回结构
所有接口统一返回：
{\n  "code": 200,\n  "msg": "success",\n  "data": {}\n}
分页返回结构
分页接口的 data 统一写成：
{\n  "list": [],\n  "total": 0,\n  "pageNum": 1,\n  "pageSize": 10\n}
5 通用状态码
200 成功\n400 参数错误\n401 未登录或 token 无效\n403 无权限\n404 数据不存在\n409 数据冲突\n500 服务器异常
6 通用字段规范
时间字段
统一格式：
2026-03-12 14:30:00
主键字段
统一使用：
id
角色字段
统一使用：
APPLICANT\nHR
岗位状态
统一使用：
OPEN\nCLOSED\nDRAFT
投递阶段
统一使用：
APPLIED\nSCREENING\nINTERVIEW\nOFFER\nREJECTED\nWITHDRAWN
